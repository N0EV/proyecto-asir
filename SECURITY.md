# Política de Seguridad (Security Policy)

> [!IMPORTANT]
> Este repositorio está destinado a un entorno académico de formación (C.F.G.S. ASIR). Para mantener la integridad del laboratorio, los reportes de seguridad se gestionan de forma privada y centralizada dentro de GitHub.

## Versiones Soportadas

El soporte y análisis de vulnerabilidades se aplica exclusivamente a las versiones estables publicadas y a las ramas de desarrollo activas del presente curso.

| Versión / Rama | Estado del Soporte | Tipo de Rama |
| :--- | :---: | :--- |
| Rama Principal (`main`) | Activo (Estable) | Producción Académica |
| Versiones estables (`v1.x.x`) | Activo (Parches de seguridad) | Versiones de Entrega / Releases |
| Ramas de Desarrollo (`dev/*`) | En Pruebas | Entorno de Laboratorio |

## Reportar una Vulnerabilidad

Si detectas un fallo en las políticas de red, credenciales de prueba expuestas o configuraciones vulnerables en los servicios del proyecto, por favor **no utilices las Issues públicas**.

Para reportar un fallo de forma totalmente confidencial y sin necesidad de correos electrónicos:

1. Dirígete a la pestaña **Security** (Seguridad) en el menú superior de este repositorio en GitHub.
2. En la barra lateral izquierda, selecciona **Vulnerabilities** (Vulnerabilidades).
3. Haz clic en el botón verde **Report a vulnerability** (Reportar una vulnerabilidad).
4. Describe el fallo detectado y los pasos necesarios para mitigarlo en el entorno de pruebas.
